## Design Context

### Users
城市规划师与数据分析师，使用该平台进行城市更新案例的知识萃取、关系建模和经验沉淀。核心场景包括：
- Schema建模：定义实体类型和关系类型
- 案例拆解：录入案例实体和关系
- 知识图谱：可视化浏览实体关系网络
- AI助手：智能问答和案例推荐

### Brand Personality
简约专业、科技理性、人文关怀。延续Apple设计语言的克制与精致，同时强调数据可视化效果和知识工作者的专业需求。

### Aesthetic Direction
- **视觉基调**：Apple风格极简设计，注重留白与层次
- **参考对象**：Apple Developer Portal, Linear, Notion
- **反模式**：避免过度装饰、复杂渐变、非功能性动画
- **主题**：浅色模式为主，CSS已支持系统级深色模式切换

### Design Principles

1. **内容优先**：知识图谱是核心，界面元素应让位于内容展示
2. **渐进披露**：通过折叠面板、侧边栏收纳复杂功能
3. **状态可见**：案例状态、图谱模式、聚焦深度始终可见
4. **一致性**：卡片圆角24px、按钮圆角12px、过渡动画0.4s cubic-bezier(0.4, 0, 0.2, 1)
5. **无障碍**：所有交互元素支持键盘导航，提供跳过链接，尊重减少动画偏好

### Design Tokens

**Colors (OKLCH)**
- Primary: `oklch(55% 0.15 250)` - 蓝紫色
- Background Primary: `oklch(99% 0.002 250)` - 近白
- Background Secondary: `#f5f5f7` - Apple灰
- Text Primary: `#1d1d1f`
- Text Secondary: `#86868b`
- Success: `oklch(60% 0.15 145)` - 绿色
- Warning: `oklch(70% 0.15 80)` - 黄色
- Error: `oklch(55% 0.2 25)` - 红色

**Typography**
- Font: Inter, -apple-system, BlinkMacSystemFont, "SF Pro Text"
- Heading: 700 weight, -0.02em letter-spacing
- Body: 400 weight, 1.6 line-height

**Spacing**
- Base unit: 0.25rem (4px)
- Card padding: 20-32px
- Gap: 8px, 12px, 24px

**Border Radius**
- Cards: 20-24px
- Buttons: 12px
- Tags: 8px

**Shadows**
- Sm: `0 1px 2px rgba(0,0,0,0.05)`
- Md: `0 4px 6px rgba(0,0,0,0.07)`
- Lg: `0 10px 15px rgba(0,0,0,0.1)`
- Hover lift: `translateY(-3px)` with shadow increase

### Adaptive Canvas Guidelines

知识图谱画布是核心交互区域，需要：

1. **响应式布局**：
   - 桌面端(1025px+)：三栏布局，左420px + 中间自适应 + 右340px
   - 平板端(900-1200px)：收窄侧边栏，左360px + 右300px
   - 移动端(<900px)：垂直堆叠，侧边栏转为顶部/底部面板

2. **画布自适应**：
   - 使用 ForceGraph2D 的 `zoomToFit()` 自适应容器
   - 节点大小根据缩放级别动态调整
   - 过滤面板、路径分析面板可折叠

3. **状态管理**：
   - 全量模式：显示所有案例实体
   - 聚焦模式：显示选中案例及其N度邻居
   - 深度控制：1-2层邻居展开

4. **交互反馈**：
   - 节点悬停：光标变化 + tooltip
   - 路径高亮：绿色节点 + 加粗连线
   - 加载状态：居中spinner
